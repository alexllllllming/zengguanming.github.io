什么是语义化标签?

简单地来说，这种标签可以清晰地告诉浏览器和开发者它所代表的含义,而不仅仅是起到表现的作用．在HTML5以前已经有不少的语义化标签了，比如常用的p,它代表的含义是一个段落;比如h1,它代表的是文章中一个最大的标题;还有blockquote,code,em等等;反之，也有一些只是作为表现的标签，例如b和i只是为了加粗和斜体．恰当地使用语义化标签的作用在于:

  1. 可以让读者更好地理解所传达的信息
  2. 帮助开发者更好地组织和维护代码

发表于 2014-08-21   |  

什么是HTML-5 Full-Screen API

不同于浏览器自身提供的网页全屏功能，Full-Screen API可以使网页中特定的元素全屏展示，或许是一张图片，或许是一段视频…重要的是，通过该api，我们能够很方便地实现全屏功能．遗憾的是，该ＡＰＩ支持新的浏览器，如果想在老的浏览器中也实现全屏，就没办法了．

来自Can I Use的浏览器兼容性数据

需要注意的是，由于该API还未进入标准化阶段，所以调用该API时还是需要加上厂商前缀(vendor prefix)．而且为了区别旧版的fullScreen API，大部分厂商(除了firefox)将fullScreen的大写S改成了小写s．

API的属性和方法

根据whatwg的草案规定:
标准的属性有两个: document.fullscreenEnabled, document.fullscreenElement.
标准的方法有两个: element.requestFullscreen(), document.exitFullscreen().
标准的事件类型也有两个: fullscreenchange, fullscreenerror．
同时，还规定:backdrop伪元素和:fullscreen伪类．

奇异值分解SVD

发表于 2014-04-25   |  

在计算机编程中，会经常用到矩阵来存储数据，而且很多时候是一个高维的稀疏的矩阵。如果说你的业务要求不需要那么高的准确度，那么我们就可以找一个方法来得到一个近似的矩阵，只抽取”重要”的部分。奇异值分解可以很好的完成这样的任务，它可以计算分解出每个特征向量的重要程度(特征值)，然后筛选掉特征值较低的部分。在推荐系统CF算法中，我们往往要构造一个很大的用户-项矩阵，如果使用svd，我们可以将它分解成三个矩阵，从中筛选掉一些特征向量，那么最终我们可以要三个较小的矩阵来近似还原之前很大的用户-项矩阵。不仅做到了降维，而且还进行了特征选择。现在对于SVD算法或者计算SVD的算法已经有很多的优化，下文只是简单初步的介绍特征向量，特征值，特征值分解，奇异值分解和计算奇异值分解的方法。
