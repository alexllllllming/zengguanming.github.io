Array类型

数组的每一项可以保存任何类型的数据，数组的大小是动态调整的。

var colors = new Array("red", "blue", "green");
//数组字面量
var colors = ["red", "blue", "green"];

如果设置某个值的索引超过了当前项数，那么数组会自动增加到该索引加一的长度。

利用length属性添加新项：

//不会像java有out of range的困扰
colors[colors.length] = "black";
colors[colors.length] = "brown";

Function 类型

函数实际上是对象，函数名实际上是指向函数对象的指针。
使用不带括号的函数名是访问函数指针，非调用函数。
没有重载。
函数的返回值可以为函数。
this引用的是函数执行环境的变量对象。

函数声明语法定义：

function sum(num1, num2){
    return num1 + num2;
}

函数表达式定义：

var sum = function(num1, num2){
    return num1 + num2;
};

函数声明与函数表达式的区别

javascript引擎解析器会先读取声明，使其在执行任何代码之前可以访问；而函数表达式，则须等到他所在的代码被执行才会被解析。
这个过程叫做函数声明提升。

alert(sum(1,2)); //work
function sum(num1, num2){
    return num1 + num2;
}
--------------------------
alert(sum(1,2)); // not work
var sum = function(num1, num2){
    return num1 + num2;
}

this引用

在我推荐的那篇文章中，举了个生动的例子,类似于”小明在路边捡到了1分钱，他很开心”。这里的代词他 类似于javascript的“代词” this ，而代词所表示的对象，需要我们从上下文去判断。因此，要理解this必须要知道如何找到所表示的对象。

有一点需要明确的是，当函数被执行时它会获得this属性，而this所指向的对象就是调用(invoke)这个函数的对象！*this*对象时在运行时基于函数的执行环境(context)绑定的。

匿名函数的执行环境具有全局性，当js运行在严格模式时，全局函数和匿名函数的 this指向undefined，不绑定任何对象。

var person = {
    firstName: "Alex",
    lastName : "Tseng",
    showFullName:function(){
        alert(this.firstName + " " + this.lastName);
    }
};
person.showFullName(); //Alex Tseng

这里this指向的是person。

文章中举了几个场景，在这些场景中this表现得有些诡异（tricky）:

当this 被当做回调函数(callback)

引用的例子：

var user = {
    data:[
    {name:"T. Woods", age:37},
    {name:"P. Mickelson", age:43}
],
clickHandler:function (event) {
    var randomNum = ((Math.random () * 2 | 0) + 1) - 1; 
    console.log (this.data[randomNum].name + " " + this.data[randomNum].age);
}
}
$ ("button").click (user.clickHandler); // Cannot read property '0' of undefined

这里看起来是user对象invoke了clickHandler函数，但是这里会产生错误且报错为属性不存在。这是因为真正的this指向其实是button对象。给出的解释是：

Note that even though we are calling the clickHandler () method with user.clickHandler (which we have to do, since clickHandler is a method defined on user), the clickHandler () method itself will be executed with the button object as the context to which “this” now refers. So this now refers to is the button ($(“button”)) object.

因为clickHandle函数时在button的上下文中被调用的，所以this指向会是button对象。
解决这样的“错误”，可以用bind(),apply(),call()函数来指定this的指向。

$("button").click (user.clickHandler.bind (user)); // P. Mickelson 43

this inside closure

    tournament:"The Masters",
    data      :[
    {name:"T. Woods", age:37},
    {name:"P. Mickelson", age:43}
],

clickHandler:function () {
    this.data.forEach (function (person) {
    console.log ("What is This referring to? " + this); //[object Window]
     console.log (person.name + " is playing at " + this.tournament);
    // T. Woods is playing at undefined
    // P. Mickelson is playing at undefined
})
}
}
user.clickHandler(); // What is This referring to? [object Window]

内部函数(inner)是不能直接访问外部中的this的，因为我们知道在作用域链中寻找标识符的规则，只要找到了第一个就不再继续向后寻找。而每个函数必然有两个属性arguments，this，所以根据寻找的规则，必然是自家的 this。

如何解决呢？在外部函数中先保存this的值，然后再在内部中引用。

var that = this;

when method is assignede to variable

看看文章中的例子就可以了，就不再详细说了，估计大家看了上述两个例子也差不多了解了。

var data = [
    {name:"Samantha", age:12},
    {name:"Alexis", age:14}
];

var user = {
    data    :[
    {name:"T. Woods", age:37},
    {name:"P. Mickelson", age:43}
],
    showData:function (event) {
        var randomNum = ((Math.random () * 2 | 0) + 1) - 1; // random number between 0 and 1
        console.log (this.data[randomNum].name + " " + this.data[randomNum].age);
    }
}
// Assign the user.showData to a variable
var showUserData = user.showData;

// When we execute the showUserData function, the values printed to the console are from the global data array, not from the data array in the user object    
showUserData (); // Samantha 12 (from the global data array)

将函数指针赋值给一个全局变量后，this的指向为windows object，与原本的意图不合。
解决方法是：

// Bind the showData method to the user object
var showUserData = user.showData.bind (user);

// Now we get the value from the user object, because the this keyword is bound to the user object
showUserData (); // P. Mickelson 43

bind()

Function Currying：

also known as partial function application, is the use of a function (that accept one or more arguments) that returns a new function with some of the arguments already set. 

 function greet (gender, age, name) {
       // if a male, use Mr., else use Ms.
       var salutation = gender === "male" ? "Mr. " : "Ms. ";

       if (age > 25) {
           return "Hello, " + salutation + name + ".";
           }
       else {
           return "Hey, " + name + ".";
            }
        }
//So we are passing null because we are not using the "this" keyword in our greet function.
var greetAnAdultMale = greet.bind (null, "male", 45);
greetAnAdultMale ("John Hartlove"); // "Hello, Mr. John Hartlove."

var greetAYoungster = greet.bind (null, "", 16);
greetAYoungster ("Alex"); // "Hey, Alex."
greetAYoungster ("Emma Waterloo"); // "Hey, Emma Waterloo."

apply() and call()

作用：

  1. borrow function
  2. 在调用函数时显示设置this值
  3. 在apply函数时进行传参 (reating and executing variadic functions)

Borrowing Functions with Apply and Call

var anArrayLikeObj = {0:"Martin", 1:78, 2:67, 3:["Letta", "Marieta",  "Pauline"], length:4 };
// Make a quick copy and save the results in a real array:
// First parameter sets the "this" value
var newArray = Array.prototype.slice.call (anArrayLikeObj, 0);

console.log (newArray); // ["Martin", 78, 67, Array[3]]

// Search for "Martin" in the array-like object
console.log (Array.prototype.indexOf.call (anArrayLikeObj, "Martin") === -1 ? false : true); // true
// Try using an Array method without the call () or apply ()
console.log (anArrayLikeObj.indexOf ("Martin") === -1 ? false : true); // Error: Object has no method 'indexOf'
 // Reverse the object:
console.log (Array.prototype.reverse.call (anArrayLikeObj));
// {0: Array[3], 1: 67, 2: 78, 3: "Martin", length: 4}

// Sweet. We can pop too:
console.log (Array.prototype.pop.call (anArrayLikeObj));
console.log (anArrayLikeObj); // {0: Array[3], 1: 67, 2: 78, length: 3}

// What about push?
console.log (Array.prototype.push.call (anArrayLikeObj, "Jackie"));
console.log (anArrayLikeObj); // {0: Array[3], 1: 67, 2: 78, 3: "Jackie", length: 4}

函数的属性arguments也是array-like对象，所以也可以对它使用apply和call。

// We do not define the function with any parameters, yet we can get all the arguments passed to it
function doSomething () {
var args = Array.prototype.slice.call (arguments);
console.log (args);
}

doSomething ("Water", "Salt", "Glue"); // ["Water", "Salt", "Glue"]

Use Apply() to Execute Variable-Arity Functions

在javascript中 Math.max()是个多参数函数：

Math.max (23, 11, 34, 56);//56

但是当我们有一个数组时，如何传给max函数呢？就可以用到apply()。

var allNumbers = [23, 11, 34, 56];
// Using the apply () method, we can pass the array of numbers:
console.log (Math.max.apply (null, allNumbers)); // 56

这里this设为null，是因为max内部并没使用到this。
