css权威指南读书笔记1

发表于 2014-04-23   |  

上下文选择符

子选择符： 标签1 > 标签2 (1为2的父元素)

紧邻兄弟选择符： 标签1 + 标签2 （紧跟在标签1后的兄弟标签）

一般兄弟选择符： 标签1 ~ 标签2 （所有的跟在标签1后的兄弟标签）

* 所有元素

p * p包含的所有元素

非子(儿子)选择符

section * a {font-size:2em;}

如果存在a是section的儿子标签，则不会被选中，但是如果section的儿子标签中包括了a标签，则该标签会被选中。(即会排除掉儿子标签)

ID和类选择符

  • 类选择符

.类名 

.special {color:red;}
  • 标签带类选择符

标签.类名 

p.special {color:red;}
  • 多类选择符

html设置多个类属性：

<p class="special unique"></p>

选择同时存在两个类的元素：

.special.unique {font-size:18em;}

(没有空格，有空格的话就变成上下文选择符了)

  • ID选择符

id可用于页内导航。

移动页面顶部的导航：

<a href="#">Back to Top</a>

# 表示告诉浏览器当前链接目标在页面内。

什么时候用ID，什么时候用类

ID的用途是在页面中唯一地标识一个元素，利用唯一的ID可以方便地定位到这个元素以及它的子元素。

类的目的是为了标识一组具有相同特征的元素。

属性选择符

基于html标签的属性选择元素。
标签名[属性名]

  • 属性值选择符

选择任何带有值为属性值的属性名的标签名。
标签名[属性名=”属性值”]

伪类

在某些事件发生时，改变某些元素的样式。

  • User Interface 伪类

:链接伪类

  • Link
  • Visited
  • Hover
  • Active (鼠标在元素上按下，还没有释放)
  • :focus伪类

表单中的文本字段在用户单击时会获得焦点。

input:focus {border:1px solid blue}
  • :target伪类

如果用户点击一个指向页面中其他元素的链接，则那个元素就是target!

维基百科在其引证中大量使用了:target伪类;如果没有:target应用突出显示，很难知道你点击的链接对应着一大堆引证中的哪一个。

  • 结构化伪类

结构化伪类是根据标记的结构来改变样式。

：first-child 和 ：last-child (是选择该元素集合中的第一个，不是元素后代的第一个)。

：nth-child(n)

li:nth-child(3) {} //选择兄弟元素中的第三个

伪元素

伪元素就是你的文档中若有实无的元素。

  • ::first-letter伪元素

    p::first-letter {font-size:300%;}

  • ::first-line

    p::first-line {font-variant:small-caps;}

  • ::before和::after

用于在特定元素前面或后面添加内容。

在boostrap中很常见，用于元素前后清空内容

基线

基线并不是汉字文字的下端沿，而是英文字母“x”的下端沿。

行距 行高

行高：相邻行文本基线间的距离 

line-height : 1.25;

行距 上一文本基线至下一文本顶线间距离

对齐方式

元素默认的垂直对齐方式为: 基线对齐。

vertical-align
语法：
baseline | sub | super | top | text-top | middle | bottom | text-bottom

字体 em

任意浏览器的默认字体高都是16px。所有未经调整的浏览器都符合: 1em=16px。
em是相对长度单位。相对于当前对象内文本的字体尺寸。
em会继承父级元素的字体大小。

border属性

默认border-style属性为none。(无边框)
常用的是： 

border-style: solid;

user-select

css3中新增，用于设置用户可以选择的内容。

  • auto——默认值，用户可以选中元素中的内容
  • none——用户不能选择元素中的任何内容
  • text——用户可以选择元素中的文本
  • element——文本可选，但仅限元素的边界内(只有IE和FF支持)
  • all——在编辑器内，如果双击/上下文点击发生在子元素上，改值的最高级祖先元素将被选中。

用于buuton中，使button中的文本元素不可选择。

width height

  • 一个元素的width是指左内边界到右内边界的距离。 （这是内容区的宽度）

如果没有显示指定元素的width，那么默认为父元素的width。

下图为继承了body的width：

一个元素的height是指上内边界到下内边界的距离。

通过3个属性设置为auto：width,margin-left,margin-right

若width为auto，则自动计算达到父元素width所需的宽度。(width默认为auto)
若设置其中两个，另外一个为auto，则该属性值会自动计算所需宽度来满足元素框的width等于父元素内容区的width。
若margin-left,margin-right都设为auto，则该元素在父元素中居中。(每个元素相对于其包含块摆放) (text-align是将内联内容居中)

元素框 = 元素width + 内边距padding

垂直居中

左右外边距为auto时会居中。
但上下外边距设为auto时，会为0，不是居中。

浮动元素

浮动元素的左右外边界受限于包含块的左右内边界。(父元素边界)
浮动元素的顶端不能比之前所有浮动元素或块级元素更高。

clear：left;

确保左边没有浮动元素。

定位 position

  • static (default)
  • relative (原本所占的空间仍保留)
  • absolute (相对于包含块定位，原先空间删除)
  • fixed (包含块为窗口本身)
  • inherit

偏移属性(offset properties)。top、right、bottom、left。
偏移是相对于元素的外边距边界。

内容溢出 overflow

  • visible (default) (元素的内容在元素框外可见)
  • hidden
  • scroll (使用滚动条来显示溢出内容)
  • auto

内容剪裁 clip

  • rect(top,right,bottom,left)
  • auto

应用于绝对定位元素。
rect的值是距元素左上角的距离，类似于元素框偏移后截取里面的内容。

元素可见性 visibility

  • visible
  • hidden (仍然影响文档布局)
  • collapse

绝对定位

绝对定位元素的包含块是最近的position值不为static的祖先元素。
