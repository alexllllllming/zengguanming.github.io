##作用域

js中的作用域是函数作用域非块作用域。

变量的作用域是从声明的开始到函数的结束。

命名函数是存在于它所声明的整个scope。(hoisting)

全局域也类似于被一个大的函数所包裹着。

函数调用与参数传递

四种途径调用函数

  1. 直接作为function被调用
  2. 作为对象的方法被调用
  3. 作为构造函数被调用
  4. 通过apply()或call()被调用

    函数参数(parameter)

传入比声明的参数数量多或少都不会报错，这是因为javascript把所有传入参数都保存在arguments中。

所有的函数调用都会默认传递两个参数：arguments、this；

this指针

this指向函数的上下文(function context)。

在java中，一个function中的this是指向该类的一个实例；在js中，this跟这个function在哪定义的，如何定义的没有关系，而是取决于这个function是如何被调用的。(function context OR invocation context)

构造函数(constructor function)

构造函数创造了上下文环境，考虑以下情况：

fullName属性还是存在于window对象中，这不是我们想要的，而构造函数可以使其存在于创建的对象中。

function Alex(){
    this.fullName = "Alex Tseng";
}
var alex = new Alex();
console.log(alex.fullName); // Alex Tseng
console.log(window.fullName); // undefined

apply() and call()

a mean to invoke a function and explicitly specify object we want as the function context.(these two methods exists in every functions.)

apply是传一个参数数组，而call是传具体的参数。

useful when invoking callback function!

inline function

内联函数名只在函数内可见。可用于递归。

  • 递归函数

hardcoded dependencies ： this，对象属性
no dependencies : inline function

function properties

用函数属性来记录计算过的值，当做cache使用！

not pollute the scope.

可以对类数组对象使用数组的方法，通过apply方法。

函数重载

In javascript, we overload function by inspecting the number and nature of the passed arguments.
利用argument参数来查看什么被传递进来，从而进行函数的重载。
方法：闭包+匿名函数

function addMethod(object, name, fn){
    var old = object[name];
    object[name] = function(){
        if (fn.length == arguments.length)
            return fn.apply(this, arguments);
        else if (typeof old == 'function')
            return old.apply(this, arguments);
    };
}

var ninjas = {
    values : ["Dean Edwards", "Sam Stephenson", "Alex Russell"]
};

addMethod(ninjas,"find",function(){
    return this.values;
});

addMethod(ninjas,"find",function(name){
    var ret = [];
    for (var i = 0; i < this.values.length; i++)
        if (this.values[i].indexOf(name) == 0)
            ret.push(this.values[i]);
    return ret;
})

arguments references a collection of all of the passed arguments.

detect function type

一些浏览器的typeof判断function结果会有点问题。
解决方法之一： 

function isFunction(fn){
    return Object.prototype.toString.call(fn) === "[object Function]";
}

closure

在函数外部作用域的所有变量，包括在inner function之后声明的变量都包含在closure中。

闭包的用处

private variable:

定义私有变量，只有在函数内部才可以访问。

function Ninja(){
    // different to this.feints = 0
    // this private variable
    var feints = 0;
    this.getFeints = function(){
        return feints;
    };

    this.feint = function(){
        feints++;
    };
}
var ninja = new Ninja();
ninja.feint(); 
ninja.getFeints(); // feints = 1
ninja.feints // undefined

在callback中访问outer data：

虽然可以直接将data定义在global scope中，但是定义在closure中可以不污染全局变量域。

Function.prototype.memoized = function(key){
    this._values = this._values || {};
    // if cached,take it. Otherwise,compute and cached.
    return this._values[key] != undefined ?
        this._values[key] :
        this._values[key] = this.apply(this,arguments);
};

Function.prototype.memoize = function(){
    var fn = this;
    return function(){
        // arguments belongs to the return function
        return fn.memoized.apply(fn,arguments);
    };
};

var isPrime = (function(num){
    var prime = num != 1;
    for (var i = 2;i < num; i++){
        if (num % i == 0){
            prime = false;
            break;
        }
    }
    return prime;
}).memoize();

console.log(isPrime(25))

function wrapping

安全地重写了已有的方法，而且还保留对原有方法的访问，以针对原有方法适合的特殊情况。

function wrap(object, method, wrapper){
    // keep the originnal method
    var fn = object[method];
    return object[method] = function(){
        return wrapper.apply(this, [fn.bind(this)].concat(
                Array.prototype.slice.call(arguments)));
    };
}

if (Prototype.Browser.Opera){
    wrap(Element.Methods, "readAttribute",function(original, elem, attr){
        return attr == "title" ?
            elem.title:
            original(elem,attr);
    });
