HTML-5 Full-Screen API

发表于 2014-08-21   |  

什么是HTML-5 Full-Screen API

不同于浏览器自身提供的网页全屏功能，Full-Screen API可以使网页中特定的元素全屏展示，或许是一张图片，或许是一段视频…重要的是，通过该api，我们能够很方便地实现全屏功能．遗憾的是，该ＡＰＩ支持新的浏览器，如果想在老的浏览器中也实现全屏，就没办法了．

来自Can I Use的浏览器兼容性数据

需要注意的是，由于该API还未进入标准化阶段，所以调用该API时还是需要加上厂商前缀(vendor prefix)．而且为了区别旧版的fullScreen API，大部分厂商(除了firefox)将fullScreen的大写S改成了小写s．

API的属性和方法

根据whatwg的草案规定:
标准的属性有两个: document.fullscreenEnabled, document.fullscreenElement.
标准的方法有两个: element.requestFullscreen(), document.exitFullscreen().
标准的事件类型也有两个: fullscreenchange, fullscreenerror．
同时，还规定:backdrop伪元素和:fullscreen伪类．

requestFullscreen()是在指定元素上调用的，而exitFullscreen是需要在document上调用,具体的细节流程可以查看whatwg;

void requestFullscreen();

使特定的元素进行全屏模式

在不同浏览器中调用时，需要加其前缀:

  • webkitRequestFullscreen
  • mozRequestFullScreen
  • msRequestFullscreen

在调用requestFullscreen()之前，需要检测浏览器是否支持fullscreen，可以通过检查document.fullscreenEnabled是否为true;

同样地，在不同浏览器调用时，需要加其前缀:

  • document.webkitFullscreenEnabled
  • document.mozFullScreenEnabled
  • document.msFullscreenEnabled

在requestFullscreen()执行过程中，会大致经历以下几步：

  1. 检查该元素是否能够全屏,需满足以下全部条件
    1.1 元素是否在document中
    1.2 元素所在document的fullscreenEnabled标志是否设置正确
    1.3 元素所在document的fullscreenElement栈是否为空
    1.4 元素没有存在iframe祖先元素

  2. 如果有错误，则会触发fullscreenerror事件;否则將该元素放到fullscreenElement栈顶,接着触发fullscreenchange事件.

同样地，若需要为fullscreenerror和fullscreenchange事件添加侦听器，也需要加其前缀:

fullscreenchange:

  • webkitfullscreenchange
  • mozfullscreenchange
  • MSFullscreenchange

fullscreenerror:

  • webkitfullscreenerror
  • mozfullscreenerror
  • MSFullscreenerror

exitFullscreen()

取消全屏时，需要在document元素上调用exitFullscreen(), 该方法在不同浏览器中命名有些许差别：

  • document.webkitExitFullscreen()
  • document.mozCancelFullScreen()
  • document.msExitFullscreen()

执行完exitFullscreen()后会触发fullscreenchange事件．

:backdrop 和 :fullscreen

:backdrop伪元素是在fullscreenElement之下的元素，可以用它来设定背景的颜色，但是到目前为止，只有在ie11上才可以设定:backdrop，其他浏览器暂时不允许改变:backdrop的样式．

比较常用的是通过:fullscreen伪类来改变元素在全屏下的样式，在不同的浏览器下也要加上对应的前缀：

  • :-moz-full-screen
  • :-webkit-full-screen
  • :-ms-fullscreen
