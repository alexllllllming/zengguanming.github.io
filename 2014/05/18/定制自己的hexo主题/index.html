定制自己的hexo主题

发表于 2014-05-18   |  

前几天突然有想去折腾下博客主题的冲动，其实之前就有想弄个自己喜欢类型的，可是有想法无干劲，现在正好趁着七分热情赶紧着手搞一搞。因为是用hexo搭的博客,所以必须先把hexo的官方文档[1]给看了,有个大概的了解就差不多了，需要重点了解的是Theme中关于layout的约定、Variables中的一些全局变量、还有一些方便的Helpers函数。

看完文档之后，大概可以着手规划怎么来做了。因为我的目的是自己用，暂时不需要高度的可定制化，所以我都取最方便、最得心应手的来写。样式是纯css写(待我css更熟之后可能会用less或其他css预处理器再重写),布局和按钮样式是用pureCSS,相比boostrap，它更轻量化，自由性更高，最重要的是它真很轻，压缩版才20KB，相对于boostrap的90KB已经很小了。排版的样式参考了typoCSS，之前用别人的主题总是感觉排版有点不舒服，所以想特意在这方面有所着重。模板是用ejs,当然你也可以用Jade其他模板语言，ejs相对来说更方便更容易上手(特别是对于有php模板、jsp模板经验的来说，大同小异)。

EJS

EJS需要学的大概就是几个标记，然后在这个标记内写你的js语句，大致的列举下几个标记。

<% code %>

在这两个标记中包围中的代码会当成是语句执行，跟下面两个标记的区别就是它不会将结果输出到页面中。

<%- code %>

与上面的区别是，左边还有个”-“号，它会将结果直接输出到页面中，不做任何处理。

<%= code %>

这个标记则是”-“号变成了”=”号，它也会将结果输出到页面中，但是在输出之前会将一些特殊的标记进行转义,例如一些html标签。
